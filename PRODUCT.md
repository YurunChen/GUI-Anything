# Product

## Register

brand

## Users

GUI-Anything is for open-source developers, agent researchers, and serious Claude Code users who run long coding sessions and need to inspect what happened after the work is done.

These users care about provenance, local control, reproducibility, and low-friction tools that do not interrupt their flow.

## Product Purpose

GUI-Anything makes agent work observable, durable, and replayable. It sits beside Claude Code, reads session JSONL, and turns a long run into a live timeline, flowchart, local wiki context, and replayable session bundle.

The website should explain the problem first: agent sessions can contain hours of exploration, but humans still need the path, evidence, pivots, and durable knowledge. Installation details are secondary to the meaning of the project.

## Brand Personality

Calm, precise, scholarly, and quietly ambitious. The brand should feel closer to an Apple research/product page than a generic developer SaaS landing page: refined typography, clear claims, generous spacing, restrained color, and a strong product artifact in the first viewport.

The voice is formal but not cold. It should respect engineers and researchers who evaluate tools carefully.

## Anti-references

Avoid noisy SaaS hero pages, generic AI productivity copy, decorative gradients, fake dashboards, heavy neon cyberpunk language, and README-like pages that lead with usage instructions.

Avoid making neobrutalism the whole identity. It can remain as a light structural accent for open-source energy, but the dominant impression should be precise, durable, and evidence-led.

## Design Principles

- Lead with the pain: agent work becomes hard to audit when the path disappears.
- Show the artifact: the demo should make the observer, flow, knowledge, and replay loop tangible.
- Keep Claude untouched: sidecar architecture is a core trust signal.
- Prefer evidence over hype: use exact concepts such as JSONL, local wiki, replay, summaries, and bundles.
- Make interaction meaningful: controls should reveal product behavior, not exist as decoration.
- Use restraint with intent: Apple-like polish, academic clarity, and open-source transparency.

## Accessibility & Inclusion

Target WCAG AA contrast for text and controls. Support reduced-motion preferences, keyboard focus, readable line lengths, and responsive layouts that preserve the product story on mobile.

Motion should clarify state changes in the demo without hiding content or requiring animation to understand the page.
